To get Font Awesome Icons working in the webflow designer, there are custom font files added in the "Project Settings" dashboard under the fonts tab.

You can copy and paste character codes for the different icons by highlighting an icon from here: https://fontawesome.com/cheatsheet

Once you have pasted an icon's character code into some text in Webflow, you can highlight the character and wrap it in a span by clicking the paintbrush icon in the little menu that appears above the highlighted text.

Then click on the span and give it a class in the styles menu at the top right, then add the font awesome font under the typography section in the styles. Make sure you choose the correct font (either regular, solid, or brands) based on which section of the cheatsheet page the icon came from.

Note for backend setup:

The actual font files for font awesome being used for webflow are not necessary after exporting, the font awesome font family css lines at the bottom of the exported css file can be removed. An import of the font awesome css file will be used instead (usually added in cms.css) @import url("https://use.fontawesome.com/releases/v5.5.0/css/all.css");

Add styles to all of the following items (without adding a class). This is so if a client adds an <h4> tag, for example, in the CMS text editor, that it has a style that matches the rest of the site. A lot of times, I just add the same styles for h3-h6 but it’s up to you. If you add a font to the <body> tag, it will effect anything that doesn’t already have a set font (add styles to the body tag by selecting any other element, then hitting the up arrow until you get to body).

Webflow doesn’t allow styles to be set for <a> tags unless they have a class, so I use the class “text-link” to represent all <a> tags. After exporting, make sure to copy styles from “.text-link” and “.text-link:hover” to all links (“a” and “a:hover”) in sitename.webflow.css. It’s important to check through all pages in the export after doing this, because it may have changed styles on menu items or buttons which use the <a> tag.
